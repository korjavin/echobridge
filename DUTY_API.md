# Duty API Contract

The EchoBridge service features a "Who is on duty" Alexa command that fetches the name of the person currently on duty from a third-party service. This document describes the expected API contract for that third-party service.

## Endpoint configuration

In order for EchoBridge to reach your service, configure the `DUTY_URL` environment variable in your `.env` file to point to your endpoint:

```env
DUTY_URL=https://your-service.com/api/duty
```

## Request Format

EchoBridge will make a standard `GET` request to your configured `DUTY_URL`.

### Authorization (Optional but recommended)

If you configure the `DUTY_SECRET` environment variable in your `.env` file, EchoBridge will sign its requests using HMAC-SHA256 to ensure they are genuinely coming from your trusted instance.

```env
DUTY_SECRET=your-shared-secret
```

When `DUTY_SECRET` is set, EchoBridge adds the following headers to the `GET` request:

*   `X-Timestamp`: A standard Unix timestamp (in milliseconds).
*   `X-Signature`: A lowercase hexadecimal HMAC-SHA256 signature generated by hashing the `X-Timestamp` value using your `DUTY_SECRET`.

#### Verifying the Signature

To verify the request on your third-party service, recalculate the HMAC-SHA256 signature using the same shared secret and the provided `X-Timestamp`, then compare it to the `X-Signature`.

Example (Node.js):

```javascript
const crypto = require('crypto');

function verifyRequest(req, secret) {
    const timestamp = req.headers['x-timestamp'];
    const signature = req.headers['x-signature'];

    if (!timestamp || !signature) return false;

    // Optional: Check if the timestamp is within an acceptable window (e.g., 5 minutes) to prevent replay attacks
    const now = Date.now();
    if (Math.abs(now - parseInt(timestamp, 10)) > 5 * 60 * 1000) {
        return false;
    }

    const expectedSignature = crypto
        .createHmac('sha256', secret)
        .update(timestamp)
        .digest('hex');

    // Use a timing-safe equal comparison
    return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expectedSignature));
}
```

## Response Format

Your endpoint should return a successful HTTP status code (e.g., `200 OK`) and should complete the request within **3 seconds** (otherwise, EchoBridge will timeout).

The response body can be provided in one of two formats:

### 1. Plain Text (Backward Compatible)

A simple string containing the name of the person on duty.

**Response Body:**
```
Alex
```

### 2. JSON Object

A JSON object containing the name, and optionally an array of chores. EchoBridge will attempt to extract the name from one of the following fields, checked in order:
1.  `name`
2.  `duty`
3.  `person`
4.  `user`

If a `chores` array is provided, EchoBridge will also read aloud each chore's details. Each chore object can optionally include:
*   `description`: What needs to be done.
*   `assignee`: Who is responsible for the chore.
*   `deadline_at`: When the chore is due.

**Response Body Example with Chores:**
```json
{
  "name": "Alex",
  "chores": [
    {
      "description": "Take out the trash",
      "assignee": "Alex",
      "deadline_at": "5:00 PM"
    },
    {
      "description": "Feed the dog"
    }
  ]
}
```

**Backward Compatible JSON Example:**
```json
{
  "duty": "Alex"
}
```

If the name cannot be extracted or evaluates to an empty string, EchoBridge will inform the user that it received invalid duty information.
